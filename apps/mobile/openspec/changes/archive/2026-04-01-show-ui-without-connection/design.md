## Context

当前 `StatusTilesGrid` 组件在没有设备连接时（`isConnected == false`）直接返回空 Widget（`SizedBox.shrink()`），导致主界面完全空白。用户无法了解应用的功能布局和预期内容。

## Goals / Non-Goals

**Goals:**
- 未连接时显示完整的界面框架和磁贴布局
- 数据用 "--" 占位符替代，让用户知道每个磁贴的用途
- 禁用交互功能（录制控制、模式切换等）避免无效操作
- 连接磁贴显示"未连接"状态，点击可弹出扫描弹窗

**Non-Goals:**
- 不修改 `CameraStatusModel` 的数据结构
- 不添加新的磁贴类型
- 不修改蓝牙连接逻辑

## Decisions

### 1. 占位符显示策略
使用 "--" 作为数据占位符，而非 "N/A" 或空字符串。理由：
- 简洁明确，用户一眼就能识别为占位状态
- 保持布局一致性，不会因为空内容导致布局错位
- 与 GPS 磁贴的 "未启用"、"获取中" 风格一致

### 2. 交互禁用方式
使用 `onTap: null` 禁用 InkWell 的点击效果，而非使用 `IgnorePointer`。理由：
- 保持视觉一致性，磁贴仍可看到
- InkWell 自动处理禁用状态下的视觉反馈
- 更简单的实现方式

### 3. 状态判断位置
在 `_buildSmallTiles` 方法内部判断连接状态，而非在 `build` 方法顶层。理由：
- 可以针对不同磁贴定制占位符内容
- 连接磁贴需要特殊处理（显示"未连接"而非 "--")
- 录制控制磁贴需要禁用而非显示占位符

## Risks / Trade-offs

[Risk] 用户可能误解 "--" 为无效数据 → Mitigation: 连接磁贴明确显示"未连接"状态，提示用户需要连接设备

[Risk] 模式选择器在未连接时显示可能引起困惑 → Mitigation: 显示默认模式但禁用滚动交互
