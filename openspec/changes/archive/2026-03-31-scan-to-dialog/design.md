## Context

当前应用使用 go_router 管理路由，`/scan` 是独立页面。连接状态磁贴 `_ConnectionTile` 点击后调用 `onDisconnect` 断开连接并跳转到扫描页面。

## Goals / Non-Goals

**Goals:**
- 扫描功能改为弹窗形式
- 点击连接状态磁贴弹出扫描弹窗
- 应用启动未连接时自动弹出扫描弹窗
- 保持自动连接和设备记忆功能

**Non-Goals:**
- 不修改扫描逻辑本身
- 不修改蓝牙连接流程

## Decisions

### 1. 使用 showModalBottomSheet 实现弹窗
**决定**: 使用 Flutter 的 `showModalBottomSheet` 实现扫描弹窗

**理由**:
- 底部弹窗符合移动端操作习惯
- 可以拖拽关闭
- 支持半展开状态
- 实现简单

**替代方案**: `showDialog` - 不适合，因为扫描列表需要较大空间

### 2. 启动逻辑调整
**决定**: 应用启动直接进入工作台，检测未连接时自动弹出扫描弹窗

**理由**:
- 简化路由结构
- 用户始终在工作台上下文中
- 弹窗可关闭，用户可以先查看界面

### 3. 连接状态磁贴行为
**决定**: 点击连接状态磁贴弹出扫描弹窗，而不是断开连接

**理由**:
- 用户可能想切换设备而不是断开
- 弹窗中可以断开连接或连接新设备
- 更符合用户预期

## Risks / Trade-offs

**[弹窗状态管理]**: 弹窗关闭后需要正确清理状态
→ Mitigation: 使用 StatefulWidget 管理弹窗内部状态

**[自动连接逻辑]**: 需要适配弹窗场景
→ Mitigation: 保持原有自动连接逻辑，在弹窗中执行
