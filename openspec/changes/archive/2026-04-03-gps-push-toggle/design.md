## Context

当前 GPS 推送功能位于独立的 GPS 设置页面 (`gps_settings_view.dart`)，包含：
- 位置信息卡片（纬度、经度、海拔、时间）
- 速度信息卡片（速度、向北/向东速度）
- 精度信息卡片（水平/垂直精度、速度精度、卫星数量）
- 自动推送开关

首页已有 `_GpsTile` 磁帖组件，当前仅显示 GPS 状态（未启用/获取中/已定位），无交互功能。

## Goals / Non-Goals

**Goals:**
- 将自动推送开关集成到首页磁帖，实现一键开关
- 移除独立的 GPS 设置页面，简化应用结构

**Non-Goals:**
- 不改变 GPS 推送的核心逻辑（GpsProvider）
- 不修改位置获取的实现

## Decisions

1. **磁帖交互设计**
   - 点击磁帖 → 切换 `autoPushEnabled` 状态
   - 显示状态：开启时显示绿色图标 + 推送状态，关闭时显示灰色图标 + "未启用"
   - 理由：保持与其他控制磁帖（如录制控制）一致的交互模式

2. **GPS 信息展示**
   - 移除独立页面后，GPS 位置信息不再单独展示
   - 首页磁帖仅显示推送开关状态，不显示具体坐标数据
   - 理由：GPS 推送是后台功能，用户关注的是开关状态而非实时坐标

3. **路由清理**
   - 移除 `/gps` 路由入口
   - 移除底部导航栏的 GPS 入口（如有）
   - 理由：功能已整合到首页，无需独立入口

## Risks / Trade-offs

- **Risk**: 用户可能需要查看详细 GPS 信息（如调试时）
  → **Mitigation**: 可在调试台页面保留 GPS 日志输出，或后续按需添加详情弹窗

- **Risk**: 移除页面可能影响现有用户的习惯
  → **Mitigation**: 首页磁帖提供更便捷的操作方式，用户体验提升
