## Context

当前 `device_scan_dialog.dart` 结构：
1. 顶部标题栏（扫描按钮、关闭按钮）
2. 自动连接横幅（"正在寻找..."）
3. 蓝牙不可用横幅
4. 设备列表（扫描结果）

用户希望：
1. 在弹层中看到已连接的设备
2. 能直接断开连接
3. 去掉"正在寻找..."的自动连接提示

## Goals / Non-Goals

**Goals:**
- 在弹层顶部显示已连接设备
- 提供断开连接按钮
- 移除自动连接横幅

**Non-Goals:**
- 不改变自动连接逻辑（保持后台自动连接）
- 不改变设备列表的其他功能

## Decisions

### 1. 已连接设备区域位置

**选择**: 放在标题栏下方、设备列表上方

**理由**:
- 已连接设备信息比扫描结果更重要，应优先展示
- 与设备列表视觉分离，区分明确

### 2. 断开连接按钮样式

**选择**: 红色文字按钮

**实现**:
```dart
TextButton(
  onPressed: () => session.disconnect(),
  child: Text(l10n.disconnect, style: TextStyle(color: theme.colorScheme.error)),
)
```

### 3. 移除自动连接横幅

**选择**: 完全移除 `_AutoConnectBanner`

**理由**: 用户反馈该提示冗余，自动连接功能仍在后台执行，只是不显示提示。

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 用户可能不知道自动连接正在进行 | 自动连接成功后弹层自动关闭，用户能看到结果 |
