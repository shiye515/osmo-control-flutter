## Context

当前 GPS 页面布局：
1. GPS 启用开关 - 独立控制 GPS 采集
2. 当前位置信息卡片 - 显示经纬度、高度、精度、时间
3. 自动推送卡片 - 开关 + 频率滑动条 (0.1Hz ~ 10Hz)
4. "立即推送"按钮 - 手动推送一次位置

用户需要先启用 GPS，再启用自动推送才能开始工作。

## Goals / Non-Goals

**Goals:**
- 简化操作流程，一键启用推送
- 移除冗余控件（GPS 开关、立即推送按钮）
- 提供固定的频率选项（2秒、5秒、10秒）

**Non-Goals:**
- 不修改 GPS 采集核心逻辑
- 不修改位置推送协议

## Decisions

### 决策 1：自动推送联动 GPS

启用自动推送时：
1. 自动调用 `setGpsEnabled(true)` 启动 GPS 采集
2. 开始按设定频率推送位置

禁用自动推送时：
1. 停止推送
2. 保持 GPS 采集状态（用户可能仍需要查看位置）

### 决策 2：频率选项

从滑动条改为 SegmentedButton 选择：

| 选项 | 间隔 | 频率 |
|-----|------|------|
| 2秒 | 2s | 0.5Hz |
| 5秒 | 5s | 0.2Hz |
| 10秒 | 10s | 0.1Hz |

UI 显示 "2秒"、"5秒"、"10秒" 更直观。

### 决策 3：简化后的页面布局

```
┌─────────────────────────────┐
│ 当前位置                     │
│ 纬度: 31.123456             │
│ 经度: 121.654321            │
│ 高度: 10.5 m                │
│ 精度: 5.0 m                 │
│ 时间: 14:30:25              │
└─────────────────────────────┘

┌─────────────────────────────┐
│ 自动推送                     │
│ [开关]                       │
│ 频率: [2秒] [5秒] [10秒]    │
└─────────────────────────────┘
```

## Risks / Trade-offs

**风险：移除独立 GPS 开关后无法单独采集位置**
- 用户启用自动推送时自动启动 GPS
- 禁用推送时 GPS 仍保持采集状态（显示当前位置）
